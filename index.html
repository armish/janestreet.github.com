The Core Suite

The Core suite of libraries is an industrial strength alternative to OCaml's standard library that was developed by Jane Street, the largest industrial user of OCaml.

Note that Core is not a compatible, drop-in replacement for the standard library. We've made different design decisions, and so code designed for the standard library would need to be adapted in order to use Core.

The Core suite has the following components:

  • Core, which is the heart of the standard library
  • Core_extended, a set of extra, less carefully tested and reviewed but still useful libraries.
  • Async, a monadic concurrency library
  • A set of syntax extensions, including sexplib and bin_prot, which extend the OCaml language itself. These are not necessary to use the rest of the libraries, but they are necessary for building them.

Core has been available for several years, but we've just moved to a new development model for our open-source libraries, and we hope this will make it easier to interact and accept patches from others. We'll have new stable releases here soon, but for now, we just have the development tree hosted here.

Interested in contributing to Core?

Great! We're happy to work with people who want to help grow Core and the associated libraries. In order to contribute to Core, though, you need to sign the Core CLA (Contributor License Agreement). You can find more about the legal details here. And you can learn more about the practicalities of contributing here.

Core is available under the Apache open-source license.
